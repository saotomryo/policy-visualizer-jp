# 税・社会保障・給付の家計インパクト 作成方針

## 方針変更メモ

初期案では `国債・金利・財政支出` を起点に家計波及を説明する構成を想定していたが、
利用者視点では因果が遠く、解釈しづらい。

そのため、実装では次の方針へ切り替える。

- `国債` は家計への直接入力変数として扱わない
- `金利` 中心の説明は外し、家計へ直接届く制度変更を扱う
- 主役は `消費税` `所得税` `社会保険料` `給付・補助`
- シナリオ名は抽象語ではなく、具体策ベースで置く
  - 消費税を一律8%
  - 消費税を一律5%
  - 食料品の消費税0%
  - 所得税の定額減税
  - 子ども・子育て支援金
  - 年収の壁対策
  - 一律給付金
  - 児童手当拡充
- シナリオ選択はページ全体共通にして、全タブで共通適用する
- `名目影響` と `実質影響` を分けて表示する

## 目的

このページは、国債残高や財政支出の大きさそのものを説明するだけでなく、
`金利変動` と `財政支出の変化` が `各家庭にどのくらい影響するか` を
家族構成、年収、年齢、住宅ローン条件ごとに可視化することを目的とする。

利用者に伝えたい中心メッセージは次の通り。

- 国の財政や国債の議論は、家計から切り離された抽象論ではない
- 金利、物価、税負担、給付、住宅ローン、預貯金収入に複数経路で波及する
- 影響は世帯属性によって大きく異なる
- 同じ政策変更でも、得をする世帯と不利になる世帯がある

## ページの基本コンセプト

テーマの軸は `金利変動 × 財政支出 × 家計属性` とする。

ページ全体は、次の順で理解できる構成にする。

1. `全体像`
国債、金利、財政支出の変化が、どの経路で家計に影響するかを整理する。

2. `家計への主な影響`
住宅ローン、物価、預金金利、税負担、給付の変化を要因別に示す。

3. `条件シミュレーター`
世帯条件を入力すると、年間と10年累積の影響額を概算表示する。

4. `モデルケース比較`
代表的な家計を並べて、誰にどの影響が大きいかを見せる。

5. `前提と限界`
これは推計モデルであり、将来予測を断定するものではないと明示する。

## 利用者に見せる主要アウトプット

条件入力後、最低限次の値を出力する。

- 世帯への年間プラス影響額
- 世帯への年間マイナス影響額
- 年間の純影響額
- 10年累積の概算影響額
- 影響内訳
  - 住宅ローン返済増減
  - 預貯金金利収入増減
  - 生活費上昇影響
  - 税・社会保険料の圧力
  - 給付・公共サービスの増減可能性

表示は `プラス` と `マイナス` を同時に見せる。
`悪化だけを煽るページ` にはしない。

## 計算ロジックの考え方

厳密なマクロ経済予測ではなく、`家計影響のモデルケース計算` として設計する。

### 1. 住宅ローン影響

- `変動金利` は政策金利や長短金利の上昇シナリオに応じて返済増を反映
- `固定金利` は新規借入時を除き影響を小さく設定
- 返済額の増減を年間影響額として表示

### 2. 預貯金・安全資産影響

- 預貯金額に対して、金利上昇分を簡易利回りとして加算
- 資産が大きい世帯ほど一部恩恵が出るようにする
- ただし、物価上昇が同時に進む場合は実質価値の目減りも別表示する

### 3. 物価上昇影響

- 世帯人数と年齢帯から基礎生活支出をモデル化
- 金利上昇や財政支出変化に連動したインフレ率シナリオを置く
- 年間生活費への上乗せ額を出す

### 4. 税・社会保険料圧力

- 国債費の増加や財政再建圧力が強まるシナリオでは、
  将来的な負担増圧力を簡易係数で反映
- 直接税額の厳密計算ではなく、`家計への追加負担圧力` として扱う

### 5. 給付・公共サービス影響

- 財政拡大型では子育てや高齢者向け給付のプラスを加味
- 緊縮型では給付縮小や自己負担増圧力を反映
- 年齢帯・家族構成ごとに感応度を変える

## シナリオ設計

シナリオはページの理解を支える中心機能なので、意味が直感的に伝わる名前にする。

### ベースシナリオ

- `現状維持`
金利も歳出も大きくは変えない

### 金利シナリオ

- `緩やかな金利上昇`
ローン負担は少し増えるが、預金金利も改善

- `急な金利上昇`
ローン負担と国債費圧力が強まり、家計への悪影響が大きい

### 財政シナリオ

- `財政拡大`
給付や公共投資は増えるが、物価や将来負担圧力にも注意が必要

- `緊縮`
物価や金利面の圧力は弱まる可能性がある一方、給付や公共サービスが縮小しやすい

## モデルケース

利用者が自分事化しやすいように、初期表示では代表的な家計を並べる。

- 若年単身・賃貸
- 子育て世帯・住宅ローンあり
- 共働き世帯・教育費負担あり
- 中高年夫婦・ローン返済終盤
- 高齢世帯・年金中心
- 資産保有が大きい世帯

## UI 方針

### 1. ファーストビュー

- 抽象的な財政説明ではなく、
  `あなたの家計では年いくら影響するか` を最初に見せる
- シナリオ切り替えを目立つ位置に置く

### 2. 結果カード

- `年間プラス`
- `年間マイナス`
- `年間純影響`
- `10年累積`

を大きく表示する。

### 3. 内訳可視化

- 棒グラフまたは積み上げ棒グラフで、影響要因の内訳を表示
- プラス要因とマイナス要因を色分けする

### 4. 世帯比較

- モデルケースごとの横並び比較を用意する
- 同じ政策でも影響の向きが違うことを分かりやすく示す

## 表現上の注意

- `国債が多いから即破綻` のような煽り表現は避ける
- `金利上昇は借り手に不利だが、預金者には一部有利` のように両面を示す
- `誰にどの経路で影響するか` を主題にする
- あくまで `概算モデル` であり、将来を断定しない

## 実装フェーズ

### フェーズ1: 簡易版

まずは次の機能だけに絞る。

- シナリオ切り替え
- 年齢帯
- 世帯年収
- 家族構成
- 住宅ローン有無
- 預貯金額
- 年間影響額
- 10年累積影響額

### フェーズ2: 拡張版

第二段階で次を追加する。

- 住宅ローン残高の詳細入力
- 固定 / 変動の差
- 給付感応度
- 税・保険料圧力の詳細化
- 高齢世帯向け年金影響
- モデルケース比較グラフ

### フェーズ3: 高度化

必要に応じて次を検討する。

- 資産配分別の影響
- インフレ感応度の個別調整
- 子育て支援・教育支援の反映
- 家計タイプ別のカスタム初期値

## 実装形式

このテーマはシミュレーション要素が強いため、外部生成HTMLとして作成してもよいが、
継続的に計算式を調整する可能性が高い。

そのため、次のどちらかで進める。

- `外部生成HTML + 直接編集`
- `テンプレートベース + ページ固有JS`

初回は開発速度を優先して、外部生成HTML方式で問題ない。
